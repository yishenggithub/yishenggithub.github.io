Java的IO系统

来自thinking in Java的笔记和总结

Posted by eason on 2018-07-12

看了thinking in Java的18章，Java I/O系统提到策略设计模式,p531。

File类
File类不仅仅只代表存在的文件或目录，也可以用File对象来创建新的目录或者不存在的整个目录路径。我们还可以查看文件的特征：大小，最后修改日期，读写，检查某个File对象代表的是文件还是目录。

输入和输出
编程语言的IO类库中常用流这一抽象概念。它代表有能力产出数据的数据源的对象或者有能力接受数据的接收端对象。“流”屏蔽了实际的I/O设备中处理数据的细节。

提到装饰者模式，p533

Note：Java中流类库让人迷惑的主要原因在于创建单一的结果流，却需要创建多个对象。

18.4.1
几乎所有原始的JavaIO流类都有相应的Reader和Writer类来提供天然的Unicode操作。然而在某些场合，向java.util.zip类库是面向字节的而不是字符串。最明智的做法是尽量尝试使用Reader和Write，知道代码无法编译。
