时隔好久更博客

Posted by eason on 2017-02-25

第二篇博文

用hexo和github搭博客是去年暑假的时候，当时还没有学node.js，github也只是听说过。花了一番力气搭好，又绑定了域名后，只是写了一篇庆祝开通的博客就搁置了，关于博客搭建的原理完全没有概念。后来域名过期，以前的域名是newyisheng.xyz，想续域名的活没有第一次买的优惠，所以换到了oneyisheng.xyz。即使以前的域名只是我一个人记得住，但还是喜欢以前的new开头。
因为新学期开始的时候对node.js感兴趣，也因为之前看springmvc的书想写打博客上来，所以才关照起了以前的博客。
因为太久没有用了，想更新却完全不知道该怎么下手，都忘了怎么操作了，git忘了，命令更是忘了，以前的文件夹在哪里也有点迷糊。然后又看了一些教程，摸索着，终于才更新了博客。

遇到的问题

  • 用node的命令行工具是git命令没有加入到path环境变量中。

  • hexo命令是在存放博客的文件夹下面用，因为hexoinit创建时在文件夹了创建了相关文件。

  • git命令行工具用hexo时找不到node，我记得以前用的就是git的命令行工具啊。

  • 把hexo s，hexo g，hexo d这些命令行弄清楚了之后，试着上传一些对以前博文的修改。传上去之后，域名打不开了。后来才发现是CNAME文件没了，得重新创建。

  • 编码问题，生成静态文件然后部署后，博文总是很奇怪。是编码的问题，用sublime就正常。

  • 还有莫名其妙多了两篇写着easomeasom的文章。可能是wps没有关。
    博客消失了，然后怎么上传都没反应，可能是用了hexo clean。

  • 使用到的工具是node.js，cd到blog2这个文件夹就可以用下面的命令操作了，编辑文本用到sublime

常用hexo命令

  • hexo new “postName” #新建文章

  • hexo new page “pageName” #新建页面

  • hexo generate #生成静态页面至public目录

  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

  • hexo deploy #部署到GitHub

  • hexo help # 查看帮助

  • hexo version #查看Hexo的版本
